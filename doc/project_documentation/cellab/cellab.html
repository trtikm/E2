E2: cellab

This library defines a parametrised model of mammalian cerebral cortex [1] called neural tissue in whole E2 project. Instances of the model (for given parameters) are primary subjects of study in this project. A parameter of the neural tissue is any datum passed to any library function constructing the model in the memory or or any algorithm passed to any library function performing a simulation step.

Structure of neural tissue

Here we specify what components the neural tissue is made of, what is their internal structure and purpose, and how they are organised in 3D space. The basic structure of the neural tissue is depicted in this picture.

We see three main components of the model: tissue, sensory cells, and synapses to muscles. The tissue is a 2D array of columns. There is X columns along the X axis and Y columns along the Y axis. Valid indices to the array represent the set {0,…,X−1}×{0,…,Y−1}. All columns in the tissue have identical structure. Each column thus contains exactly the same count C of tissue cells. Due to clarity of the picture these cells are not depicted in the picture. Nevertheless, they are aligned along the columnar axis C. A tissue cell is uniquely identified in its column by its columnar coordinate which from the set {0,…,C−1}. A tissue cell is uniquely identified in the issue by three coordinates along the axes X,Y, and C. In other words, tissue cells are regularly distributed in the left-handed 3D Cartesian space [2]. The model allows to split tissue cells in a column into K different kinds {0,…,K−1}. We can see in the picture that first N0 tissue cells in each column are of the kind 0, next N1 tissue cells are of the kind 1, and so on. Observe that C = N0 + … + NK−1. The model does not define interpretations of kinds of tissue cells. This is left for a user of the model. For example, a user may interpret a kind as an identifier of an algorithm simulating certain behavioral property of a cell. The structure of a column is actually more complex than described so far.
