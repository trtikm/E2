E2: cellab

This library defines a parametrised model of a mammalian cerebral cortex [1] called neural tissue in the whole E2 project. Instances of the model (for given parameters) are primary subjects of the study in this project. A parameter of the neural tissue is any datum passed to any library function constructing the model in the memory or any algorithm passed to any library function performing a simulation step.

Structure of neural tissue

Here we specify what components the neural tissue is made of, what is their internal structure and purpose, and how they are organised in 3D space. The basic structure of the neural tissue is depicted in this picture.

We see three main components of the model: tissue, sensory cells, and synapses to muscles. The tissue is the component of the model where the main computation is performed. Sensory cells and synapses to muscles allow for information exchange between the tissue and an external environment (surrounding the whole model). Namely, sensory cells allows for receiving information from the environment, while synapses to muscles allows for sending information from the tissue to the environment. We describe all tree components of the model in more details in the following three subsections.

Tissue

The tissue is a 2D array of columns. There is X columns along the X axis and Y columns along the Y axis. Valid indices to the array represent the set {0,…,X−1}×{0,…,Y−1}. All columns in the tissue have identical structure. Each column thus contains exactly the same count C of tissue cells. Due to clarity of the picture these cells are not depicted in the picture. Nevertheless, they are aligned along the columnar axis C. A tissue cell is uniquely identified in its column by its columnar coordinate taken from the set {0,…,C−1}. A tissue cell is uniquely identified in the issue by three coordinates along the axes X, Y, and C. In other words, tissue cells are regularly distributed in the left-handed 3D Cartesian space [2]. The model allows to split tissue cells in a column into K different kinds {0,…,K−1}. We can see in the picture that first N0 tissue cells in each column are of the kind 0, next N1 tissue cells are of the kind 1, and so on. Observe that C = N0 + … + NK−1. The model does not define interpretations of kinds. This is left for a user of the model. For example, the user may interpret a kind as an identifier of an algorithm simulating certain behavioral property of a cell.

Sensory cells

Sensory cells allows for delivery of information form the external environment to the tissue. They are split into S different sensory kinds {0,…,S−1}. The model does not define interpretations of these kinds. This is left for a user of the model. For example, the user may interpret them as an assignment of sensory cells to particular sense organs like eye, ear, nose, etc. The user is responsible for encoding of information captured by the organs in the corresponding sensory cells.

Due to purely technical reasons (which will be presented later here) we align sensory cells with the columnar axis C of the tissue. It means that each sensory cell has a unique coordinate along the axis. We can see in the picture that the first sensory cell (depicted at the top) is associated with the first free coordinate C along the axis (the columnar axis is depicted on the left of the list of the sensory cells). Note though that sensory cells do not have coordinates along X and Y axes of the tissue.

Observe further that the number of sensory cell of a sensory kind i is represented by a number NK+i. This notation follows from the technicals reasons mentioned above. It allows us to encapsulate both kinds of cells (tissue and sensory) and their counts under one representation accessed using a single mechanism. The last sensory cell (depicted at the bottom) is assigned the highest valid coordinate CS along the columnar axis C. Clearly, CS = C + NK+0 + … + NK+S−1 = N0 + … + NK+S−1.

Synapses to muscles

Synapses to muscles allows for delivery of information form the tissue to the external environment. They are split into U different muscle kinds {0,…,U−1}. The model does not define interpretations of these kinds. This is left for a user of the model. For example, the user may interpret them as an assignment of synapses to particular muscles. The user is responsible for decoding of information stored in the synapses and updating the environment according to an effect of the received information on corresponding muscles. Note that the update of the environment may have an effect on sense organs which it turn implies a request for an update of sensory cells.

We can see in the picture above the number of synapses to muscles of a muscle kind i is represented by a number Mi. The total number of synapses to muscles in the model is thus M0 + … + MU−1.
