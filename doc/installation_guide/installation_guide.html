Software dependences

The project depends on several software packages. You need to install them first. We further recommend to install them in the order presented here.

Required packages for all users:

  • g++ version 4.8.1 or later. It is the C++ compiler of the GCC collection of compilers. Note that E2 project requires a support of C++11 standard. TODO: specify a terminal command to install the package!!!
  • CMake version 2.8 or later. It is a build system for C++ program. We recommend to install also Qt GUI application cmake-gui. You can install both CMake and its GUI either in Ubuntu Software Center by searching for a string cmake gui or in the terminal using a command:
    • sudo apt-get install cmake cmake-gui
  • Python development 64 bit version 3.4 or later. We emphasise that the development version is required. TODO: specify a concrete package name for apt-get as an example!!! Let <python-install-dir> represents a complete path to the directory where you installed the Python. If you use Ubuntu Software Center or apt-get for the installation, then the installation directory should normally be /usr/lib/python<python-version>, where <python-version> is a concrete version of Python (two numbers connected by the dot) you installed (e.g. 3.4). E2 uses several Python scripts to manage simulations and testing.
  • Boost C++ libraries version 1.59.0 or later. Unpack a content of the downloaded package. Let <boost-install-dir> represents a complete path to the directory with the unpacked content. Now open the terminal (e.g. by pressing Ctrl+Alt+T) and type there the following three commands:
    • cd <boost-install-dir>
    • ./bootstrap.sh --with-python=python<python-version>
    • ./b2 --build-dir="<boost-install-dir>/build/gcc" address-model=64 toolset=gcc stage
  • Eigen linear algebra library version 3.2.5 or later. Let <eigen-install-dir> represents a complete path to the directory with the unpacked content. The Eigen library is header only, no building is necessary. So, the unpacking of the downloaded package completes the installation.
  • Qt5 framework 64 bit version 5.4.1 or later. Make sure you install also Qt Creator IDE version 3.4.2 or later (it is a common part of the framework's package). Getting E2 project

There are two main ways how to get the project. Either by a direct download of a compressed package or by cloning project's repository using Git.

In the first case you only need to unpack the downloaded package into some directory on the hard drive. For future reference let <E2-install-dir> represents a complete path to the directory containing the unpacked project files.

The second case (cloning the project's repository) is a preferred and strongly recommended way of getting the E2 project. We describe two ways how you can do that. First, using SmartGit and second in the terminal.

Cloning project's repository in terminal

First you need to create some directory on your hard drive, where you want to install the project. Let us denote the complete path of that directory <E2-install-dir>. Next, open the terminal (e.g. by pressing Ctrl+Alt+T) and type there the following two commands:

  • cd <E2-install-dir>
  • git clone https://github.com/trtikm/E2.git .

Note that the ending dot '.' in the second command is its necessary part. So, do not forget it!

Building E2 in Qt Creator

Start the Qt Creator. First we configure Kits. In the menu navigate to Tools>Options>Kits and select Desktop (Default) kit under Manual kits. Now select the following compiler and debugger (it is very likely, that they are already selected):

  • GCC (x86 64bit in /usr/bin)
  • System GDB at /usr/bin/gdb

Then confirm your choice by clicking on the button OK.

Next in the menu navigate to File>Open File or Project and open there the file

  • <E2-install-dir>/CMakeLists.txt

In the subsequent dialog specify the following build directory:

  • <E2-install-dir>/build/<build-type>

where <build-type> can be either Debug or Release. Choose one according to your preferences. (You may later create another configuration for the other choice. You will be then able to switch between the configurations any time). Finally, confirm your chice by clickin on the button Configure project.

Now switch to Projects tab (shortcut Ctrl+5, or click to the 5th icon from the top in the vertical bar appearing at the very left of the application window). Into the edit box titled CMake arguments write the following line of variable definitions:

  • -DCMAKE_BUILD_TYPE=<build-type> -DE2_BUILD_ALSO_TESTS=<yes-no> -DBOOST_INSTALL_DIR=<boost-install-dir> -DEIGEN_INSTALL_DIR=<eigen-install-dir> -DPYTHON_LIBRARY=<python-install-dir>

where possible values for <yes-no> are Yes or No. Then click on the button Details inside the section Build Steps and amongst available targets choose install.

Finally, go to the menu and select there Build>Run CMake. This completes the configuration. Check the output produced by the CMake in the General Messages window (shortcut Alt+6) that no error occurred in the process. Normally, the output should be without any errors. But if there are some then use the output to resolve them.

You can build the project by pressing Ctrl+B or my selecting a menu item under the Build menu (there are several possibilities for your choice). All resulting binaries can then be found in the directory:

  • <E2-install-dir>/dist

Building E2 in terminal

First we have to use cmake-gui application to generate build scripts of the native Makefiles build system. So, open the terminal (e.g. by pressing Ctrl+Alt+T) and type there a command:

  • cmake-gui

Into the edit-box Where is the source code insert the path:

  • <E2-install-dir>

Then into the edit-box Where to build the binaries insert the path:

  • <E2-install-dir>/build/<build-type>

where <build-type> can be either Debug or Release.

Then press the button Configure. If the build directory does not exist yet (which is very likely to be the case), then let CMake to create it for you. In the subsequent dialog select the following:

  • In the drop-down box called Specify the generator for this project choose the option Unix Makefiles.
  • And select the radio-button Use default native compilers.

It is very likely that both options are already selected in the dialog. Confirm your selection by clicking on the button Finish.

The process should terminate with errors. That is expected, because we have not specified all necessary information yet. Namely, we have to specify the following variables shown in the application with the red background (the red background of a variable indicates a necessity to specify or verify its value):

  • CMAKE_BUILD_TYPE     Specify your preferred <build-type> (i.e. either Debug or Release).
  • E2_BUILD_ALSO_TESTS     Write there Yes, if you want to build also tests, and No otherwise.
  • BOOST_INSTALL_DIR     Specify the complete path <boost-install-dir>.
  • EIGEN_INSTALL_DIR     Insert the complete path <eigen-install-dir>.
  • PYTHON_LIBRARY     Write there the complete path <python-install-dir>.

You can ignore all other variables, if there are any (and even if they are with the red background).

Now repeat the configuration process (i.e. clicking on the button Configure) several time (typically up to 4 times), until all red backgrounds disappears. Then also check the output in the edit-box bellow the list of variables, that the configuration was successful. In case of errors (it is unlikely to happen) use the output messages to resolve them.

Now click on the button Generate to generate the native build scripts. Then you can terminate the cmake-gui application.

We are back in the terminal and we can finally build the project using the following two commands:

  • cd <E2-install-dir>/build/<build-type>
  • make install

All resulting binaries can then be found in the directory:

  • <E2-install-dir>/dist

Configuring E2 in PyCharm

The configuration is extremely simple, because both Python scripts and HTML pages are used as they are, i.e. no compilation is needed. So, start PyCharm and in the menu navigate to File>Open... item. Then put the path <E2-install-dir> into the edit-box in the opened dialog and confirm your choice by clicking on the button OK. That is all.
